# HVAC Salesforce Project Standards

## 1) Repository & Folder Structure
All Salesforce work for this portfolio must live under:

- `salesforce/hvac/`

Recommended structure:
- `salesforce/hvac/force-app/main/default/`
  - `classes/`
  - `triggers/`
  - `lwc/`
  - `objects/`
  - `flows/`
  - `permissionsets/`
  - `profiles/` (avoid if possible; prefer permission sets)
  - `labels/`
  - `staticresources/`
- `salesforce/hvac/manifest/`
- `salesforce/hvac/docs/`

**Rule:** Do not place Salesforce metadata outside `salesforce/hvac/`.

---

## 2) Naming Rules (Global)
### 2.1 File Name Prefix
**All file names must start with `HVAC`** (where applicable).

Examples:
- ✅ `HVAC_PricingService.cls`
- ✅ `HVAC_QuoteTrigger.trigger`
- ✅ `HVAC_salesDashboard.js` (if you enforce file prefixing in LWC files)
- ✅ `HVAC_Installation_Job__c.object-meta.xml`
- ❌ `PricingService.cls`
- ❌ `QuoteTrigger.trigger`

> Note: Salesforce DX generates some filenames based on metadata (e.g., LWC bundle files). For those, enforce the prefix at the **component/bundle name** level, and accept the generated internal filenames.

### 2.2 Metadata Component Prefix
**All custom components (Apex, Flows, LWCs, custom objects) must start with `HVAC` in their API/component names.**

Examples:
- Apex Class: `HVAC_PricingService`
- Trigger: `HVAC_QuoteTrigger`
- Custom Object: `HVAC_Installation_Job__c`
- Permission Set: `HVAC_Sales_User`
- Named Credential (if used): `HVAC_ERP_NamedCredential`

---

## 3) Flows (Mandatory Rule)
**Flow API names must start with: `HVAC_FLW_`**

Format:
- `HVAC_FLW_<Object>_<Timing>_<Purpose>`

Timing values (recommended):
- `BeforeSave`
- `AfterSave`
- `Scheduled`

Examples:
- ✅ `HVAC_FLW_Lead_AfterSave_AssignByRegion`
- ✅ `HVAC_FLW_InstallationJob_AfterSave_ActivateWarranty`
- ✅ `HVAC_FLW_Warranty_Scheduled_ExpiryNotifications`
- ❌ `Lead_Assignment_Flow`
- ❌ `FLW_Lead_Assign`

---

## 4) Apex (Classes & Triggers)
### 4.1 Apex Classes
**Prefix:** `HVAC_`

Recommended suffixes (choose the right one):
- `*Service` (business logic): `HVAC_PricingService`
- `*Handler` (trigger orchestration): `HVAC_QuoteTriggerHandler`
- `*Selector` (query layer): `HVAC_InventorySelector`
- `*Client` (callouts): `HVAC_ErpOrderClient`
- `*Controller` (LWC endpoints): `HVAC_SalesDashboardController`
- `*Util` (small helpers only): `HVAC_StringUtil`

### 4.2 Apex Triggers
**Prefix:** `HVAC_`

Format:
- `HVAC_<Object>Trigger`

Examples:
- ✅ `HVAC_QuoteTrigger`
- ✅ `HVAC_OrderTrigger`
- ✅ `HVAC_InstallationJobTrigger`

---

## 5) Custom Objects & Fields
### 5.1 Custom Objects
**Prefix:** `HVAC_` and must end with `__c`

Examples:
- ✅ `HVAC_Installation_Job__c`
- ✅ `HVAC_Warranty__c`
- ✅ `HVAC_Service_Request__c`
- ✅ `HVAC_AMC_Contract__c`
- ✅ `HVAC_Inventory__c`

### 5.2 Custom Fields
Recommended pattern:
- `HVAC_<Field_Name>__c` on custom objects
- On standard objects, still use `HVAC_` prefix for custom fields.

Examples:
- ✅ `HVAC_Region__c`
- ✅ `HVAC_Discount_Percentage__c`
- ✅ `HVAC_Installation_Cost__c`
- ✅ `HVAC_ERP_Invoice_Number__c`

---

## 6) Lightning Web Components
### Component (bundle) naming
**LWC bundle name must start with `hvac` (lowercase)** to match JS conventions, while still satisfying the HVAC prefix intent.

Recommended pattern:
- `hvacSalesDashboard`
- `hvacTechnicianConsole`

Folder names (generated by SFDX) will match the bundle name.

If you want strict alignment with “HVAC prefix everywhere”, use:
- `hVACSalesDashboard` is ugly; best practice is lowercase `hvac` for LWCs.

---

## 7) Validation Rules, Permission Sets, Reports (Optional but recommended)
### Validation Rules
- `HVAC_VR_<Object>_<RuleName>`
Example: `HVAC_VR_Quote_Discount_Max_20`

### Permission Sets
- `HVAC_<Role>_PS`
Example: `HVAC_Sales_PS`, `HVAC_ServiceTech_PS`

---

## 8) Definition of Done (DoD) for Any Story
A Jira story can be moved to Done only when:
- Naming follows this standard (prefixes + formats)
- Component works in org (tested manually)
- No errors in debug logs
- Changes retrieved into repo under `salesforce/hvac/`
- Commit message is clear (e.g., `Add HVAC_FLW_Lead_AfterSave_AssignByRegion`)

---

## 9) Exceptions
- Salesforce-generated internal filenames inside LWC bundles are acceptable as-is.
- Managed package components are not renamed.
- If a Salesforce limitation prevents a prefix, document the exception in `/salesforce/hvac/docs/exceptions.md`.
